# LarvaERP Landing Page

Modern, clean landing page for LarvaERP built with **Vite + React + Tailwind CSS**, deployed on **Cloudflare Pages** with automatic deployments from GitHub.

## Quick Facts

- **Live URL:** https://larvaerp.larvalabs-apps.store
- **Portal URL:** https://larvaerp-portal.larvalabs-apps.store
- **Repository:** https://github.com/SMLarvaLabs/larvaerp-landing-page
- **Deployment:** Auto-deploys on push to `main` branch
- **Build:** `npm run build` → `dist/` folder

## Architecture

```
GitHub (main branch)
    ↓ (webhook on push)
Cloudflare Pages
    ↓ (auto build & deploy)
https://larvaerp.larvalabs-apps.store (static assets)
```

The landing page is **static HTML/React** and does NOT include the app itself. The actual LarvaERP application runs on a separate portal subdomain via Cloudflare Tunnel.

## Portal Integration

### Landing Page Links
Two CTA buttons link to the portal:
1. **Hero section** (`src/components/Hero.jsx`) - "Login to Portal →" button
2. **CTA section** (`src/components/CTA.jsx`) - "Launch LarvaERP Now" button

Both point to: **https://larvaerp-portal.larvalabs-apps.store**

### Why Sibling Subdomain?
The portal uses `larvaerp-portal.larvalabs-apps.store` (sibling) instead of `portal.larvaerp.larvalabs-apps.store` (nested) due to **Cloudflare SSL certificate generation limitations on nested subdomains pointing to tunnel endpoints**. See "Important Notes" below.

## Important Notes

### ⚠️ Cloudflare SSL Limitation
**Problem:** Nested subdomains (e.g., `portal.larvaerp.larvalabs-apps.store`) pointing to Cloudflare Tunnel endpoints via CNAME cause TLS handshake failures because Cloudflare's Universal SSL doesn't generate certificates for these patterns.

**Solution:** Use sibling subdomains (e.g., `larvaerp-portal.larvalabs-apps.store`) instead. Works flawlessly.

**If adding a new feature/portal:**
- Always use sibling subdomain structure: `appname-portal.yourdomain.store`
- NOT nested: `portal.appname.yourdomain.store`

### Styling
- Uses **Tailwind CSS** for all styling
- Custom classes in `src/index.css`:
  - `.gradient-text` - Blue-to-purple gradient for headings
  - `.glass-effect` - Frosted glass background effect
  - `.smooth-transition` - Smooth hover transitions

### Component Design
- Each component is self-contained and focused
- No shared state between components (no Redux/Context needed)
- All links are absolute URLs (no client-side routing)
- Responsive design: Mobile-first approach with `sm:`, `md:`, `lg:` Tailwind breakpoints

## Common Tasks

### Update a CTA Button Link
Edit the button in the relevant component and push:
```bash
# In Hero.jsx or CTA.jsx, change the href attribute
<a href="https://new-url.com">Button Text</a>

git add src/components/Hero.jsx
git commit -m "Update button link"
git push origin main
```

### Change Hero Text/Images
Edit `src/components/Hero.jsx` - all changes are in JSX, no external config needed.

### Add a New Section
1. Create `src/components/NewSection.jsx` with a React component
2. Import it in `src/App.jsx`
3. Add it to the render order
4. Push to GitHub

Example:
```jsx
// src/components/NewSection.jsx
export default function NewSection() {
  return (
    <section className="py-20 px-4">
      <div className="max-w-4xl mx-auto">
        <h2 className="text-4xl font-bold">New Section</h2>
        {/* Content here */}
      </div>
    </section>
  )
}
```

Then in `src/App.jsx`:
```jsx
import NewSection from './components/NewSection'

export default function App() {
  return (
    <>
      {/* ... other components ... */}
      <NewSection />
      {/* ... */}
    </>
  )
}
```

## Troubleshooting

### Page not updating after push?
1. Check GitHub Actions to see if build succeeded
2. Clear browser cache (Ctrl+Shift+Del or Cmd+Shift+Del)
3. Wait 2-3 minutes for CDN propagation
4. Check Cloudflare Pages dashboard → Deployments tab

### Build fails locally?
```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
npm run build
```

### Styling not applying?
- Ensure `tailwind.config.js` includes all template paths: `./src/**/*.{jsx,js}`
- Run `npm run build` and check CSS is in `dist/` folder
- Clear browser cache
